Axis Range Calculation

This document describes how the length of a tilted axis is calculated.

Figure 9 shows an oblique coordinate system with the red x-axis and green y-axis, an underlying Cartesian system with light grey axes, and an example viewbox with the corners B1 to B4. These corners define the extensions of the oblique coordinate system with the corners D1 to D4.

The oblique coordinate system is defined by the angles η and θ and a transformation matrix:

obliqueSystem.setAttribute( "transform", `matrix( 
	${ Math.cos( eta ) }, 
	${ Math.sin( eta ) }, 
	${ -Math.sin( theta ) }, 
	${ Math.cos( theta )  }, 
	0, 
	0 )` ) ;

with η and θ in radian units. To find the coordinates of the viewpoint corners in the oblique coordinate system, we use the inverse matrix to transform their Cartesian coordinate into the oblique coordinate system:

const im = obliqueSystem.transform.baseVal[ 0 ].matrix.inverse( );
const B1 = new DOMPoint( vbox.left, vbox.bottom ).matrixTransform( im );
const B2 = new DOMPoint( vbox.right, vbox.bottom ).matrixTransform( im );
const B3 = new DOMPoint( vbox.right, vbox.top ).matrixTransform( im );
const B4 = new DOMPoint( vbox.left, vbox.top ).matrixTransform( im );

The direction of the independent axis determines the viewbox corners that limit the range for the dependent axis. For example, if the x-axis points to the 1st quadrant, B2.y and B4.y provide the limits for the y-axis. If the x-axis points to the 2nd quadrant, the B1.y and B3.y provide these limits.

The following table relates primary axis orientation to corner points:

Quadrant Points
1 B2 B4
2 B1 B3
3 B2 B4
4 B1 B3

The orientation of the secondary axis relates corner points to its min and max values: If the (secondary) y-axis points to the 1st or 2nd quadrants, y-min = B4.y and and y-max = B2.y. If the y-axis points to the 3rd or 4th quadrants, y-min = B2.y and and y-max = B4.y.

All these relationships can be condensed into the following table:

Dependent axis
Q1 Q2 Q3 Q4
Independent axis Q1 B4,B2 B2,B4
Q3
Q2 B3,B1 B1,B3
Q4

The qudrant number 0…3 is returned by the following function:

function quadrant( phi ) {
	let q = Math.floor( phi / 90 ) % 4 ;
	return q < 0 ? q += 4 : q ;
	}

Note that you have to add 90° to θ to get the right quadrant number for the y-axis:

qx = quadrant( eta );
qy = quadrant( theta + 90 );

The range of visible coordinates Ci are defined by these corner points Bi:

C1.x = B1.x ;
C2.y = B2.y ;
C3.x = B3.x ;
C4.y = B4.y ;
C1.y = C2.x = C3.y = C4.x = 0 ;

Range End Points

As already mentioned, the viewbox rectangle in combination with the axis angles η and θ defines the visible coordinate range for the axes. This is shown for the x-axis in Figure 10. The y-axis orientation selects the points B1 and B2 as construction points for the lower and upper limits for the x-axis.

In general:

Angle range Quadrant Corners
0 90 1 top left lower right
90 180 2 top right lower left
180 270 3 bottom right upper left
270 360 4 bottom left upper right

The qudrant number 0…3 is returned by the following function:

function quadrant( phi ) {
	let q = Math.floor( phi / 90 ) % 4 ;
	return q < 0 ? q += 4 : q ;
	}

For example,

if 0≤φ<90, the quadrant is 0
if 360≤φ<450, the quadrant is 3
if 0>φ≥-90, the quadrant is 3.

The x-axis direction defines on which sides D1 and D2 go. D2 goes to the side to which the x-axis points to, D1 goes to the opposite side. More specifically, the angle between the x and y-axes:

δ=(90°+θ−η)mod360°

If δ>180°, then D1 and D2 must be swapped.

Using Trigonometry

Let's add some more details to the diagram figure 12:

Depending on the angle θ, the point B designates different corners of the viewbox. In this case, where the y-axis points down right into the first SVG quadrant and −90°>θ≤0°, it is the upper right corner. In general:

the segment BD is parallel to the oblique y-axis, and represents the largest visible x-coordinate in the oblique system. To find the point D, we draw a parallel to the oblique y-axis through point A to the intersection with the obliqe x-axis.

Then we can express the segment BA=BD−DA.

Step 1 finds the length of c=BA in the triangle ABC, which has a right angle at the corner C. The length of the sides a and b can be derived from the SVG viewbox properties x, y, width and height:

C=(viewbox.x+viewbox.width,0) D=(viewbox.x+viewbox.width,−viewbox.y)

Then we can apply the Pythagorean theorem and get

c=a2+b2

Now we can focus on the triangle ABD to find its sides e and f. There is no right angle inside, so we must use the rule of sines:

sinδc = sinεe = sinφf

Since c is known, we can solve for e and f:

e= csinε sinδ f= csinφ sinδ

The angle δ can be obtained from the axis rotation angels. We can see at the origin that

δ =90°−η−(−θ) =90°−η+θ =cos(θ−η)

Note that the angle θ is drawn counter-clockwise in figure 9, so it is a negative value, and thus we must add θ, not subtract.

The angle ε lies opposite to the side e and can be calculated as

ε=β+(−θ) = β−θ = atanba−θ

Similarly, the angle φ lies opposite to the side f and can be calculated as

φ=α+η =atanab+η

Injected into equations (1) and (2), we get

e= csin(atanba−θ) cos(θ−η) f= csin(atanab+η) cos(θ−η)

Since we are only interested in the length of the length of the x-axis, equation (3) is the solution, the value of e is rounded up to the next higher step value.
