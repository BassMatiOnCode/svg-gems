Oblique Coordinate Systems

In an oblique coordinate system, the axes are at arbitrary angles, rotated around their origin. The angle between them can differ from 90 degrees, creating a non-orthogonal oblique coordinate system. The tilted grid lines create a diamond-shaped (rhombic) pattern as shown in demo-02.htm:

η: θ:

Such a coordinate system can be created graphically by rotating the axes about the origin (0,0) by arbitrary angles, then drawing the grid lines equidistant and parallel to them. Arithmetically, we can use a linear tranformation to transfrom the coordinates from the oblique coordinate system (red) into the underlying Cartesian coordinate system of SVG (grey).

Let's have a look at figure 1:

The oblique coordinate system has the axes u and v with the basis vectors u⇀=OP and v⇀=OQ. The u axis goes from top left to bottom down and is rotated clockwise by η=20° from the horizontal, the v-axis goes from top right to bottom left and is tilted by η=30° from the vertical.

The Cartesian (SVG) coordinate system is shown in grey. Its x-axis with the basis vector x⇀=OXis oriented horizontally from left to right, its y-axis with the basis vector y⇀=OY goes vertically from top to bottom.

Note that all four basis vectors end on the unit circle with r=1 around the origin O=(0,0), they all have the length 1 in both coordinate systemss.

The red points have unit coordinates (0,1), (1,0), (1,1) in the oblique coordinate system. The searched transformation function must yield their (grey) Cartesian coordinates.

To find the coefficients a and b, refer to figure 2:

The point X lies on the unit circle. We find point P as the intersection of the oblique x-axis with the unit circle. The point Q is the orthogonal projection of P onto the Cartesian x-axis, therefore we have a right triangle OPQ with the right angle at the point Q.

Then we can find

a=rcosη b=rsinη

Since P lies on the unit circle, r=1, and we get

a=<cosη b=sinη

Step 2 determines the transformation coefficients c and d. The construction in figure 3 is similar to the one above:

We find the point Y=(0,1) on the Cartesian y-axis. This point lies on the unit circle around the origin O=(0,0). Then we construct point R as intersection of the oblique v-axis with the unit circle and project it orthogonally onto the y-axis, which gives us the point S. This creates a triangle with the points ORS with a right angle at S.

Since r=1, we can express the sides c and d as

c=−sinη d=cosη

Note that c is negative since a positive angle η produces negative x-values (c goes in the negative x-direction).

These results can be combined to obtain the transformation matrix

M= [ a c b d ] = [ cosη −sinη sinη cosη ]

This matrix can be used to obtain the Cartesian coordinates (x,y) of any point (u,v) in the oblique coordinate system:

x=ucosη−vsinθ y=usinη+vcosθ

These equations represent our transformation function. They can be found in the transform() function in demo 2. Every point in the oblique coordinate system must be passed through this transform() function in order to get its equivalent Cartesian SVG coodinates.

Transformation Matrix

SVG natively supports a matrix transform. The attribute transform="matrix(a, b, c, d, e, f)" introduces a transformation matrix of the following form:

M= ( a c e b d f 0 0 1 )

The coefficients a to d correspond to our tranformation matrix from above, the parameters e and f control a linear translation in x and y direction.

M= ( cosη −sinθ e sinη cosθ f 0 0 1 )

Therefore, the parameters are

a=cosη b=sinη c=−sinη d=cosη e=0 f=0

An example:

<g	id="obliqueCoordinateSystem" 
	transform="matrix( 0.94, 0.34, -0.5, 0.87, 0, 0)" >
	...
	</g>

Preventing Distortion 1

A matrix transformation, applied to the coordinate system group, distorts every geometrical aspect of elements inside that group. This might undesirable, for example for axis tick labels or dots that designate points, as demonstrated in demo 3:

η: θ:

The advantage of the matrix transformation is that the grid does not have to be recreated as in demo 2, it is sufficient to change the transformation matrix when the axis angles change:

demo3.contentWindow.setGridMatrix( );

One method to prevent distortion is to pull these elements out of that group and positition them manually with the help of the transform function:

<g id="obliqueCoordinateSystem" transform="matrix( ... )" >
	<!-- Elements in this group will be distorted -->
	</g>
<g id="annotations">
	<!-- Manually positioned elements go here. -->
	</g>

The elements in the annotations group should not be positioned with attributes. Instead, add a coords attribute that contains a space-separated list of numbers that will be used for positioning:

<text coords="50 -18" >1</text>
<text coords="100 -18" >2</text>
<text coords="150 -18" >3</text>

Depending on the element type, these numbers will be used to position the element

function positionElements ( ) {
	for ( const e of svgRoot.querySelectorAll( "[coords]" )) {
		const points = transform( e.getAttribute( "coords" ).split( " " ));
		switch ( e.tagName ) {
		case "circle" :
			e.setAttribute( "cx" , points[ 0 ] );
			e.setAttribute( "cy" , points[ 1 ] );
			break ;
		case "text" :
			e.setAttribute( "x" , points[ 0 ] );
			e.setAttribute( "y" , points[ 1 ] );
			break ;
		case "rect" :
			e.setAttribute( "x" , points[ 0 ] );
			e.setAttribute( "y" , points[ 1 ] );
			e.setAttribute( "width" , points[ 2 ] );
			e.setAttribute( "height" , points[ 3 ] );
			break ;
			}
		}
	}

Preventing Distortion 2

If elements are positioned manually to prevent distortion, they must be pulled out of the oblique coordinate system and put in a container with cartesian coordinates – which makes creation and maintenance of an SVG document harder. In this chapter, we will leave these elements in their natural context and apply a compensation matrix on them.

The linear part of this matrix is the inverse transformation matrix from above. Since linear transformations also change the position, we will use the e and f coefficients to compensate this translation.

Finding the Coefficients a and b

To find the inverse matrix geometrically, we will express the Cartesian basis vectors x^ and y^ in terms of the oblique basis vectors u^ and v^.

We begin with the inverse rotation matrix. In the first step, we find a linear combination of u^= and v^ for x^:

x^=au^+bv^

Figure 6 illustrates that:

Just like above, grey lines represent the Cartesian system and red lines represent the oblique coordinate system. The line segment OX represents the Cartesian basis vector x^, so

|OX|=x=1

Now we construct the line XP as a parallel to the oblique v-axis through point X=(1,0), which intersects with the oblique u-axis in the point P. This creates the triangle OPX with the sides a, b and x.

The triangle is not a right trianlge, so we must use the rule of sines to find the length of sides:

sinγx = sinαa = −sinηb

The value of b is negative because we are traveling from O to P to X, and the last part is in the opposite direction of the basis vector v^.

Since x=1, we can solve for a and b:

a= sinα sinγ b=− sinη sinγ

It is obvious from the angles at point X, that we can express α in terms of η:

α=90°−θ

Since sin(90°−θ)=cos(−θ)=cosθ, we get

a= cosθ sinγ

The sum of interior angles in a triangle is 180°, so we can express γ in terms of η and η:

η+α+γ=180° γ=180°−η−90°+θ γ=90°−η+θ

Finding the Coefficients c and d

In the second step, we will express the Cartesian basis vector y^ as linear combination of the oblique basis vectors u^ and v^:

y^=cu^+dv^

So let's have a look at figure 7:

Since |OY| represents the Cartesian basis vector y^,

y=|OY|=1

We create a triangle OQY by drawing a parallel to the oblique v-axis through Y and obtain Q at the intersection with the obliqe u-axis. Then we call OQ=c and QY=d. Their lengths represent the corresponding transformation matrix entries c and d.

We apply the law of sines and find

sinθc = sinδd = sinεy

Since γ and ε are supplementary angles, their sines are identical:

sinε=sinγ

an we will substitute γ for ε below.

Since y=|OY|=1, we can solve for c and d:

c= sinθ sinγ d= sinδ sinγ

With

δ=90°−η sin(90°−η)=cosη

follows

d= cosη sinγ

Compensating Translation

A transformation, appliend on an element, rotates, scales or skews around the coordinate system's origin (0,0), not around the element's position. Therefore the element is displaced from the point P1=(x1,y1) to the point P2=(x2,y2>) when the compensation matrix is applied. To compensate this displacement, we need to find the inverse displacement vector .

[ e f ] = P1−P2

To find P2, we apply the comensation matrix Mc to P1

[ x2 y2 ] = [ ac bd ] [ x1 y1 ] = [ ax1+cy1 bx1+dy1 ]

Inserted into equation (10):

e = x1−(ax1+cy1) = x1−ax1−cy1 f = y1−(bx1+dy1) = y1−bx1−dy1

Grouping the terms:

e=(1−a)x1−cy1 f=(1−d)y1−bx1

Preventing Distortion 3

The idea is to change the positioning method by setting the element's position attributes to zero and add a transform="matrix( a, b, c, d, x, y )" attribute:

<text x="0" y="0" transform="matrix(a, b, c, d, x, y )">tickmark label</text>

where a to d represent the inverse rotation matrix coefficients.

The element location is now the oblique coordinate system origin, where it is "undistorted", then moved (translated) to the desired location (x,y). This matrix is constructed in two steps. First, the e and f parameters are set up in drawGrid() with a transform="translate(x,y)" attribute, where x and y are the oblique target coordinates:

function drawGrid ( ) {
	...
	points[ 0 ] = x ;
	points[ 1 ] = y ;
	const text = createText( [0, 0] , ( i / gridConfiguration.xstep ).toFixed( 0 ), 
		{ compensateDistortion : true , transform: `translate(${ points[ 0 ] },${ points[ 1 ] })` }
		) ;
	...
	compensateDistortion( );
	}

After creation and later, whenever the oblique coordinate axis angles are changed, compensateDistortion() is called:

function updateGridMatrix( ) {
	setGridMatrix( );
	compensateDistortion( );
	}

This function creates the compensation matrix coefficients:

function compensateDistortion( ) {
	const k = Math.sin( 90 * degree - gridConfiguration.xangle + gridConfiguration.yangle );
	const cs = [   // rotation compensation matrix coeffitients
		Math.cos( gridConfiguration.yangle )/ k ,
		-Math.sin( gridConfiguration.xangle ) / k , 
		Math.sin( gridConfiguration.yangle ) / k ,
		Math.cos( gridConfiguration.xangle ) / k
		] ;

A loop runs through the elements to be compensated, and initializes/updates the transformation matrix coefficients a to d:

	for ( const e of obliqueCoordinateSystem.querySelectorAll( "[ compensateDistortion ]" )) {
		const matrix = e.transform.baseVal[0].matrix;
		matrix.a = cs[ 0 ];
		matrix.b = cs[ 1 ];
		matrix.c = cs[ 2 ];
		matrix.d = cs[ 3 ];
		}

This leaves the coefficients e and f, which contains the element's position in oblique coordinates, unchanged.

Transformation Matrix

The transform="matrix(a, b, c, d, e, f) attribute introduces a transformation matrix of the following form:

M= ( a c e b d f 0 0 1 )

If η rotates the X axis and θ rotates the Y axis, then the matrix is

M= ( cosη −sinθ e sinη cosθ f 0 0 1 )

Therefore, the parameters are

a=cosη b=sinη c=−sinθ d=cosθ e=0 f=0

The demo-02.htm has the r-axis tilted counter-clockwise by the angle η and the s-axis clockwise by the angle θ. To get from the coordinates r and s to the coordinates x and y in the orthogonal SVG coordinate system, we have to convert them as follows:

x=rcosη−ssinθ y=rsinη+scosθ

The transform() function implements this mapping and also take the inverted SVG Y axis into account:

function transform( r, s ) {
	const x = r * Math.cos( eta ) - s * Math.sin( theta );
	const y = r * Math.sin( eta ) + s * Math.cos( theta );
	return { x : x , y : −y } ;	
	}

The function is used to transform the RS coordinates to the SVG XYcoordinates. For example, the Y axis is create as follows:

const p1 = transform( i, -limit );
const p2 = transform( i, +limit );
line( p1, p2, color, gridGroup );

Panning can be done in one of two ways:

  1. Pan the viewport (camera): This is what we did in the previous demo. It moves the viewport (window) horizontally and/or vertically over the scene. This can be done by modifying the SVG viewBox parameters.
  2. Pan the coordinates: This pans the viewport parallel to the grid lines. The viewBox changes will have to respect the tilt angles of the axes.

Usually, option 1 feels more natural to the user.
