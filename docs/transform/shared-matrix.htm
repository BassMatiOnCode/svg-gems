Sharing a Matrix between SVG Elements

Can an SVGTransform object be shared between SVGElements, so that they all react to changes in that shared transform matrix? Short answer: No. Each element has its own transform instance.

There are three methods to get close to the goal:

  1. Place the elements inside a group (G) element and apply the transform on the group element.
  2. Create a matrix and distribute (duplicate) it to the target elements
  3. Create the matrix as CSS custom property and use a CSS transform that references the custom property.

Method 2

 

To make the code working, we need to add an SVGTransform to the elements:

// Creates the first transform list entry in the elements 
// with an identity matrix.
const transform = svgRoot.createSVGTransformFromMatrix( 
	svgRoot.createSVGMatrix( ));
for ( const e of svgRoot.children ) 
	e.transform.baseVal.appendItem( transform );

Then we can update the transformation matrix:

function updateAngles( phi ) { 
	// Replaces the first transform list entry with 
	// an updated transform object.
	const transform = svgRoot.createSVGTransformFromMatrix( 
		svgRoot.createSVGMatrix( ).rotate( phi ));
	for ( const e of svgRoot.children )
		e.transform.baseVal[ 0 ] = transform ;
	}

Method 3

Create the following CSS code:

:root {
	--shared-matrix: matrix(1,0,0,1,0,0);
	}

.shared {
	transform: var(--shared-matrix);
	transform-box: fill-box;
	transform-origin: center;
	}

Then update the matrix with the following script:

document.documentElement.style.setProperty (
	"--shared-matrix", "matrix(1,0,0,1,50,20)" ) ;

All elements selected by the .shared{ } rule will reflect the change immediately.

 

Things to remind:

  1. CSS styles override element attributes. You can retrieve the attribute, but the transform will be overridden.
  2. CSS styles overrice the element.transform list. SVGTransform elements in this list have no effect.

This means that SVG elements cannot have individiually composed transform lists.
